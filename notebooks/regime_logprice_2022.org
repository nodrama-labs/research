* Description

Phase 0.5a of the heavy-tail HMM dashboard plan, following the negative
result in [[file:regime_student_t_2022.org][regime_student_t_2022.org]] and the literature review in
[[file:literature_review.org][literature_review.org]]. Single-knob change: replace the daily log-return
observation with a *scaled log-price* observation (paper 2,
Pakštaitė et al. 2025). Same window, same comparison frame, same EM
driver, same restart strategy.

Window: 2021-06-01 → 2023-06-30 (~2 years wrapping the 2022 bear).
Asset: BTCUSDT daily candles.
Observation: ~obs_t = (log(close_t) − μ_logp) / σ_logp~ over the full
window.
Models compared: Gaussian and Student's-t HMMs at K ∈ {2, 3}.

Pass criterion (restricted to 2022 only), same as Phase 0 but evaluated
per (K, family):
- Median bear-posterior ≥ 0.6
- Regime flips ≤ 6 over 2022
- Gap vs Gaussian (at same K) ≥ 0.15 OR flip-count reduction ≥ 30 %

Hypothesis: log-price is non-stationary in level but stationary-ish
when scaled to the test window; "bear" maps to "price below scaled
mean" and "bull" maps to "price above scaled mean", which gives the
latent state something cumulative to bind to rather than the
fat-tailed-walk-near-zero that defeated Phase 0.

Risk: the EM may detect "first half of window vs second half" as the
latent state (a non-stationarity artefact rather than a bull/bear
distinction). Diagnostic: if the K = 2 fit gives a transition matrix
with p_ii ≈ 1 (essentially no switching) and the bear-state covers
exactly the contiguous 2022-mid block, that's the failure mode.

** Forward-backward in log-space

#+BEGIN_SRC jupyter-python :session llms :kernel llms_kernel :async yes :exports both
def forward_backward(log_pi, log_A, log_B):
    """
    log_pi: (K,)   log initial-state probs
    log_A:  (K,K)  log transition matrix, A[i,j] = P(s_{t+1}=j | s_t=i)
    log_B:  (T,K)  log emission probs, B[t,i] = log p(o_t | s_t=i)
    returns: log_gamma (T,K), log_xi (T-1,K,K), log_likelihood (scalar)
    """
    T, K = log_B.shape
    log_alpha = np.full((T, K), -np.inf)
    log_beta = np.full((T, K), -np.inf)

    log_alpha[0] = log_pi + log_B[0]
    for t in range(1, T):
        log_alpha[t] = logsumexp(log_alpha[t - 1][:, None] + log_A, axis=0) + log_B[t]

    log_beta[T - 1] = 0.0
    for t in range(T - 2, -1, -1):
        log_beta[t] = logsumexp(log_A + log_B[t + 1][None, :] + log_beta[t + 1][None, :], axis=1)

    log_likelihood = logsumexp(log_alpha[T - 1])
    log_gamma = log_alpha + log_beta - log_likelihood
    log_xi = (
        log_alpha[:-1, :, None]
        + log_A[None, :, :]
        + log_B[1:, None, :]
        + log_beta[1:, None, :]
        - log_likelihood
    )
    return log_gamma, log_xi, log_likelihood
#+END_SRC

#+BEGIN_SRC jupyter-python :session llms :kernel llms_kernel :async yes :exports both
def gaussian_log_emissions(x, mu, sigma):
    z = (x[:, None] - mu[None, :]) / sigma[None, :]
    return -0.5 * np.log(2 * np.pi) - np.log(sigma)[None, :] - 0.5 * z * z


def student_t_log_emissions(x, mu, sigma, nu):
    K = len(mu)
    out = np.empty((len(x), K))
    for i in range(K):
        out[:, i] = student_t.logpdf(x, df=nu[i], loc=mu[i], scale=sigma[i])
    return out
#+END_SRC

** Init (shared) and transition / initial-state M-step

#+BEGIN_SRC jupyter-python :session llms :kernel llms_kernel :async yes :exports both
def kmeans_init(x, K, rng, jitter=0.0):
    km = KMeans(n_clusters=K, n_init=10, random_state=int(rng.integers(1 << 31)))
    labels = km.fit_predict(x.reshape(-1, 1))
    mu = np.array([x[labels == i].mean() for i in range(K)])
    sigma = np.array([x[labels == i].std(ddof=0) + 1e-6 for i in range(K)])
    pi = np.array([(labels == i).mean() for i in range(K)])
    if jitter > 0:
        mu = mu + rng.normal(0, jitter * x.std(), size=K)
    # Sticky transition prior so EM has to do real work to break the diagonal.
    # For scaled log-price the regimes should be much longer than for log-returns,
    # but a 0.95 diagonal is already permissive enough to let EM concentrate further.
    A = np.full((K, K), 0.05 / (K - 1)) if K > 1 else np.ones((1, 1))
    np.fill_diagonal(A, 0.95)
    return mu, sigma, pi, A


def m_step_init_trans(log_gamma, log_xi):
    log_pi = log_gamma[0] - logsumexp(log_gamma[0])
    log_A = logsumexp(log_xi, axis=0) - logsumexp(log_xi, axis=(0, 2))[:, None]
    return log_pi, log_A
#+END_SRC

** Gaussian M-step

#+BEGIN_SRC jupyter-python :session llms :kernel llms_kernel :async yes :exports both
def m_step_gaussian(x, log_gamma):
    gamma = np.exp(log_gamma)
    Nk = gamma.sum(axis=0)
    mu = (gamma * x[:, None]).sum(axis=0) / Nk
    diff_sq = (x[:, None] - mu[None, :]) ** 2
    sigma = np.sqrt((gamma * diff_sq).sum(axis=0) / Nk) + 1e-6
    return mu, sigma
#+END_SRC

#+RESULTS:

** Student's-t M-step (ECME)

#+BEGIN_SRC jupyter-python :session llms :kernel llms_kernel :async yes :exports both
def m_step_student_t(x, log_gamma, mu_prev, sigma_prev, nu_prev,
                     nu_lo=3.0, nu_hi=30.0):
    gamma = np.exp(log_gamma)
    K = gamma.shape[1]

    z2 = ((x[:, None] - mu_prev[None, :]) / sigma_prev[None, :]) ** 2
    w = (nu_prev[None, :] + 1.0) / (nu_prev[None, :] + z2)
    gw = gamma * w

    Nk = gamma.sum(axis=0)
    Sw = gw.sum(axis=0)

    mu = (gw * x[:, None]).sum(axis=0) / Sw
    diff_sq = (x[:, None] - mu[None, :]) ** 2
    sigma = np.sqrt((gw * diff_sq).sum(axis=0) / Nk) + 1e-6

    z2_new = ((x[:, None] - mu[None, :]) / sigma[None, :]) ** 2
    w_new = (nu_prev[None, :] + 1.0) / (nu_prev[None, :] + z2_new)
    const = ((gamma * (np.log(w_new) - w_new)).sum(axis=0) / Nk)
    const = const + digamma((nu_prev + 1) / 2.0) - np.log((nu_prev + 1) / 2.0)

    nu = np.empty(K)
    for i in range(K):
        c = const[i]
        def f(v, c=c):
            return np.log(v / 2.0) - digamma(v / 2.0) + 1.0 + c
        try:
            lo_val, hi_val = f(nu_lo), f(nu_hi)
            if lo_val * hi_val > 0:
                nu[i] = nu_lo if abs(lo_val) < abs(hi_val) else nu_hi
            else:
                nu[i] = brentq(f, nu_lo, nu_hi, xtol=1e-3)
        except Exception:
            nu[i] = nu_prev[i]
    return mu, sigma, nu
#+END_SRC

** Generic Baum-Welch driver + restart wrapper

#+BEGIN_SRC jupyter-python :session llms :kernel llms_kernel :async yes :exports both
@dataclass
class HMMFit:
    log_pi: np.ndarray
    log_A: np.ndarray
    mu: np.ndarray
    sigma: np.ndarray
    nu: np.ndarray
    log_gamma: np.ndarray
    log_likelihood: float
    n_iter: int
    converged: bool
    family: str
    K: int


def viterbi(log_pi, log_A, log_B):
    T, K = log_B.shape
    delta = np.full((T, K), -np.inf)
    psi = np.zeros((T, K), dtype=np.int64)
    delta[0] = log_pi + log_B[0]
    for t in range(1, T):
        scores = delta[t - 1][:, None] + log_A
        psi[t] = scores.argmax(axis=0)
        delta[t] = scores.max(axis=0) + log_B[t]
    path = np.empty(T, dtype=np.int64)
    path[-1] = int(delta[-1].argmax())
    for t in range(T - 2, -1, -1):
        path[t] = psi[t + 1, path[t + 1]]
    return path


def baum_welch(x, K, family, rng, max_iter=1000, tol=1e-5, nu0=8.0, jitter=0.0):
    mu, sigma, pi, A = kmeans_init(x, K, rng, jitter=jitter)
    log_pi = np.log(pi + 1e-12)
    log_A = np.log(A + 1e-12)
    nu = np.full(K, nu0) if family == "student_t" else np.full(K, np.inf)

    prev_ll = -np.inf
    converged = False
    it = 0
    for it in range(max_iter):
        if family == "gaussian":
            log_B = gaussian_log_emissions(x, mu, sigma)
        else:
            log_B = student_t_log_emissions(x, mu, sigma, nu)
        log_gamma, log_xi, ll = forward_backward(log_pi, log_A, log_B)
        log_pi, log_A = m_step_init_trans(log_gamma, log_xi)
        if family == "gaussian":
            mu, sigma = m_step_gaussian(x, log_gamma)
        else:
            mu, sigma, nu = m_step_student_t(x, log_gamma, mu, sigma, nu)
        if abs(ll - prev_ll) < tol:
            converged = True
            it += 1
            break
        prev_ll = ll
    return HMMFit(log_pi, log_A, mu, sigma, nu, log_gamma, ll, it, converged, family, K)


def fit_with_restarts(x, K, family, n_restarts=5, master_seed=RNG_SEED, **kw):
    best = None
    master = np.random.default_rng(master_seed)
    for r in range(n_restarts):
        rng = np.random.default_rng(master.integers(1 << 31))
        jitter = 0.0 if r == 0 else 0.15
        try:
            fit = baum_welch(x, K, family, rng, jitter=jitter, **kw)
        except Exception as e:
            print(f"  restart {r} ({family}, K={K}) failed: {e}")
            continue
        if best is None or fit.log_likelihood > best.log_likelihood:
            best = fit
    return best

For K = 2 the canonical ordering is (bear, bull). For K = 3 it is
(bear, ranging, bull). Both use ascending μ — for scaled log-price this
is "below mean" → "near mean" → "above mean", which is the
economically natural mapping (and avoids the Phase-0 trap where μ-sort
on log-returns bound "bear" to crash days).

def regime_posterior(fit):
    order = np.argsort(fit.mu)
    gamma = np.exp(fit.log_gamma)
    return gamma[:, order], order

#+RESULTS:

* Pass criterion — 2022 only

Same flip-counting rule as Phase 0: Viterbi for primary, argmax-of-
marginal as a noise sanity check. The comparison limb of crit3 is
within-K: Student's-t must beat Gaussian *at the same K* by ≥ 0.15 on
the bear-posterior gap, or reduce flips by ≥ 30 %. This keeps the
apples-to-apples promise intact.

#+BEGIN_SRC jupyter-python :session llms :kernel llms_kernel :async yes :exports both
def viterbi_states_canonical(fit, order):
    if fit.family == "gaussian":
        log_B = gaussian_log_emissions(obs, fit.mu, fit.sigma)
    else:
        log_B = student_t_log_emissions(obs, fit.mu, fit.sigma, fit.nu)
    raw = viterbi(fit.log_pi, fit.log_A, log_B)
    remap = np.empty_like(order)
    for canonical_idx, raw_idx in enumerate(order):
        remap[raw_idx] = canonical_idx
    return remap[raw]


def count_flips(state_seq):
    return int((np.diff(state_seq) != 0).sum())


# Compute Viterbi states for all four fits.
viterbi_states = {key: viterbi_states_canonical(fit, orders[key])
                  for key, fit in fits.items()}


def evaluate_2022(K):
    """Pass-criterion evaluation for a given K, comparing Student-t vs Gaussian."""
    g_post = posteriors[("gaussian", K)][idx_2022]
    t_post = posteriors[("student_t", K)][idx_2022]
    g_st = viterbi_states[("gaussian", K)][idx_2022]
    t_st = viterbi_states[("student_t", K)][idx_2022]

    # Bear is canonical state 0 in both K=2 and K=3 (ascending-μ sort).
    g_bear_med = float(np.median(g_post[:, 0]))
    t_bear_med = float(np.median(t_post[:, 0]))

    g_flips = count_flips(g_st)
    t_flips = count_flips(t_st)
    g_flips_arg = int((np.diff(g_post.argmax(axis=1)) != 0).sum())
    t_flips_arg = int((np.diff(t_post.argmax(axis=1)) != 0).sum())

    gap = t_bear_med - g_bear_med
    flip_reduction = (g_flips - t_flips) / g_flips if g_flips > 0 else 0.0

    crit1_g = g_bear_med >= 0.6
    crit1_t = t_bear_med >= 0.6
    crit2_g = g_flips <= 6
    crit2_t = t_flips <= 6
    crit3 = (gap >= 0.15) or (flip_reduction >= 0.30)

    def verdict(crit1, crit2, crit3_or_solo):
        score = sum([crit1, crit2, crit3_or_solo])
        return "PASS" if score == 3 else ("BORDERLINE" if score == 2 else "FAIL")

    return {
        "K":                           K,
        "g_bear_median":               g_bear_med,
        "t_bear_median":               t_bear_med,
        "g_viterbi_flips_2022":        g_flips,
        "t_viterbi_flips_2022":        t_flips,
        "g_argmax_flips_2022":         g_flips_arg,
        "t_argmax_flips_2022":         t_flips_arg,
        "gap":                         gap,
        "flip_reduction_pct":          100 * flip_reduction,
        "crit1_g_bear_med_ge_0.6":     crit1_g,
        "crit1_t_bear_med_ge_0.6":     crit1_t,
        "crit2_g_flips_le_6":          crit2_g,
        "crit2_t_flips_le_6":          crit2_t,
        "crit3_gap_or_flipreduction":  crit3,
        # Verdict measures the Student-t vs Gaussian comparison (the original Phase-0 question).
        "verdict_student_t_vs_g":      verdict(crit1_t, crit2_t, crit3),
        # Also flag whether the Gaussian *alone* would have passed crits 1 & 2 — useful because
        # at log-price scale the Gaussian itself may already separate the bear.
        "gaussian_alone_passes_1_2":   crit1_g and crit2_g,
    }


for K in (2, 3):
    print(f"\n=== K = {K} ===")
    result = evaluate_2022(K)
    for k, v in result.items():
        print(f"  {k:34s} {v}")

** Hypothesis under test

Phase 0.5a was set up to test whether the *single* change from
log-return to scaled log-price as the observation — paper 2's choice,
endorsed by our own Phase-0 root-cause diagnosis that "regime
structure lives at the cumulative-price level, not the per-day-return
level" — turns the Phase-0 FAIL into a PASS on the same 2022-bear
question, same window, same EM machinery.

** What we found

The hypothesis survives. On BTCUSDT daily 2021-06-02 → 2023-06-30
(759 observations) with scaled log-price ~obs_t = (log p_t − μ̂_logp) / σ̂_logp~
as the observation:

|---------------------------------|-----:|--------:|-----:|

The bull leg is correctly identified, 2022 is dominantly bear (55 %),

** Why it works — root cause

The Phase-0 diagnosis predicted this. The latent regime structure
we want to detect — "2022 was a sustained bear" — is a statement about
*cumulative path* (price trajectory below its 2-year reference) not
*per-day distribution* (sign and magnitude of single-day returns). At
the per-day-return scale, 2022 and 2021 are statistically
indistinguishable (σ ≈ 3.5 % daily, drift differential ≈ 0.4 %/day
inside one-year sampling noise). At the scaled-log-price scale, 2022
is unambiguously and persistently below zero. EM has a feature to bind
to, and it does.

Two corollaries the experiment makes precise:

1. *Student-t is moot at this scale.* In all four fits ν̂ saturates at
   the upper bound (30), meaning the EM prefers the
   indistinguishable-from-Gaussian limit. Scaled log-price has thin
   tails because the heavy tails of daily returns are *averaged out*
   when we look at the level rather than the difference. The original
   Phase-0 question "do Student-t emissions cleanly identify 2022
   where Gaussian mis-classifies" has the trivial answer: at this
   feature choice, Gaussian alone is sufficient and Student-t adds
   nothing.

2. *K = 2 is enough.* K = 3 produces a coherent (bear, ranging, bull)
   decomposition (the ranging state usefully catches the 2023-H1
   recovery), but K = 2 already passes the original pass criterion
   with median bear-posterior = 1.00 and 1 Viterbi flip. The plan's
   commitment to K = 3 was overspecified for the 2022-bear question;
   it's the right K for a longer-window analysis with multiple cycle
   transitions.

** Caveats

Three things worth flagging before declaring this a clean win:

1. *In-sample z-scaling.* ~scaled_log_price~ uses the mean and std of
   ~log_price~ over the full 2-year window, so it has hindsight built
   in. For a forward-looking deployment we'd need either a rolling
   z-score (paper 2's 100-day rolling fit setup does this implicitly)
   or detrended log-price (subtract a 200-day MA). The 2022-bear
   detection should survive both alternatives because the bear is
   ~5 months long and the rolling baseline lags accordingly, but it's
   worth confirming.

2. *"Bear" = "below 2-year mean", not "going down".* Because the
   observation is a level, the model labels every below-baseline day
   "bear" regardless of direction. That's why 2023-H1 (recovery from
   the cycle low) is still 41 % bear in the K=3 fit — price is rising
   but still below the 2-year mean. This is the correct labelling
   under the chosen feature; it's *not* the same labelling a momentum
   or drawdown feature would produce. For the Phase-0 question
   ("identify the 2022 bear") this is fine. For a tactical regime
   classifier ("am I in a bear *right now*") it's not.

3. *Lookahead in window choice.* The window 2021-06 → 2023-06 was
   chosen to "wrap the 2022 bear" — i.e. its endpoints already encode
   the answer. Phase 1+ should re-run on rolling windows that don't
   know where the bear is.
