#+RESULTS:

* HMM core — shared forward-backward + Baum-Welch

Both Gaussian and Student's-t share the same forward-backward, the same
init (k-means on log-returns for μᵢ, per-cluster std for σᵢ, ν₀=8), the
same convergence rule, and the same restart strategy. Only the emission
log-pdf and the M-step for emission parameters differ.

** Forward-backward in log-space

#+BEGIN_SRC jupyter-python :session llms :kernel llms_kernel :async yes :exports both
def forward_backward(log_pi, log_A, log_B):
    """
    log_pi: (K,)   log initial-state probs
    log_A:  (K,K)  log transition matrix, A[i,j] = P(s_{t+1}=j | s_t=i)
    log_B:  (T,K)  log emission probs, B[t,i] = log p(o_t | s_t=i)
    returns: log_gamma (T,K), log_xi (T-1,K,K), log_likelihood (scalar)
    """
    T, K = log_B.shape
    log_alpha = np.full((T, K), -np.inf)
    log_beta = np.full((T, K), -np.inf)

    log_alpha[0] = log_pi + log_B[0]
    for t in range(1, T):
        # log_alpha[t,j] = logsumexp_i(log_alpha[t-1,i] + log_A[i,j]) + log_B[t,j]
        log_alpha[t] = logsumexp(log_alpha[t - 1][:, None] + log_A, axis=0) + log_B[t]

    log_beta[T - 1] = 0.0
    for t in range(T - 2, -1, -1):
        log_beta[t] = logsumexp(log_A + log_B[t + 1][None, :] + log_beta[t + 1][None, :], axis=1)

    log_likelihood = logsumexp(log_alpha[T - 1])

    log_gamma = log_alpha + log_beta - log_likelihood

    # log_xi[t,i,j] = log_alpha[t,i] + log_A[i,j] + log_B[t+1,j] + log_beta[t+1,j] - log_likelihood
    log_xi = (
        log_alpha[:-1, :, None]
        + log_A[None, :, :]
        + log_B[1:, None, :]
        + log_beta[1:, None, :]
        - log_likelihood
    )
    return log_gamma, log_xi, log_likelihood
#+END_SRC

#+RESULTS:

** Emission log-pdfs

#+BEGIN_SRC jupyter-python :session llms :kernel llms_kernel :async yes :exports both
def gaussian_log_emissions(x, mu, sigma):
    """x: (T,), mu/sigma: (K,) -> (T,K) log p(x_t | state=i)."""
    # standard Gaussian log-pdf
    z = (x[:, None] - mu[None, :]) / sigma[None, :]
    return -0.5 * np.log(2 * np.pi) - np.log(sigma)[None, :] - 0.5 * z * z


def student_t_log_emissions(x, mu, sigma, nu):
    """x: (T,), mu/sigma/nu: (K,) -> (T,K) log p(x_t | state=i)."""
    # scipy.stats.t parametrizes by df=nu, loc=mu, scale=sigma
    K = len(mu)
    out = np.empty((len(x), K))
    for i in range(K):
        out[:, i] = student_t.logpdf(x, df=nu[i], loc=mu[i], scale=sigma[i])
    return out
#+END_SRC

#+BEGIN_SRC jupyter-python :session llms :kernel llms_kernel :async yes :exports both
def kmeans_init(x, K, rng, jitter=0.0):
    """K-means initialisation, optionally jittered for restart diversity.

    1D K-means is essentially deterministic on a fixed dataset, so naive
    restarts converge to the same basin. We add a small Gaussian
    perturbation to the cluster means (scaled by σ(x) × jitter) so
    each restart explores a different neighbourhood.
    """
    km = KMeans(n_clusters=K, n_init=10, random_state=int(rng.integers(1 << 31)))
    labels = km.fit_predict(x.reshape(-1, 1))
    mu = np.array([x[labels == i].mean() for i in range(K)])
    sigma = np.array([x[labels == i].std(ddof=0) + 1e-6 for i in range(K)])
    pi = np.array([(labels == i).mean() for i in range(K)])
    if jitter > 0:
        mu = mu + rng.normal(0, jitter * x.std(), size=K)
    # sticky transition prior so EM has to do real work to break the diagonal
    A = np.full((K, K), 0.05 / (K - 1))
    np.fill_diagonal(A, 0.95)
    return mu, sigma, pi, A


def m_step_init_trans(log_gamma, log_xi):
    """Closed-form M-step for π and A."""
    log_pi = log_gamma[0] - logsumexp(log_gamma[0])
    log_A = logsumexp(log_xi, axis=0) - logsumexp(log_xi, axis=(0, 2))[:, None]
    return log_pi, log_A
#+END_SRC

** Gaussian M-step

#+BEGIN_SRC jupyter-python :session llms :kernel llms_kernel :async yes :exports both
def m_step_gaussian(x, log_gamma):
    gamma = np.exp(log_gamma)             # (T,K)
    Nk = gamma.sum(axis=0)                # (K,)
    mu = (gamma * x[:, None]).sum(axis=0) / Nk
    diff_sq = (x[:, None] - mu[None, :]) ** 2
    sigma = np.sqrt((gamma * diff_sq).sum(axis=0) / Nk) + 1e-6
    return mu, sigma
#+END_SRC

#+RESULTS:

** Student's-t M-step (scale-mixture-of-Gaussians)

Uses the standard scale-mixture EM update: each observation gets a
posterior precision weight ~w_{t,i} = (νᵢ + 1) / (νᵢ + zᵢ²)~ where
~zᵢ = (xₜ - μᵢ)/σᵢ~. Then μ and σ get weighted-MLE updates, and ν
solves the digamma equation numerically.

#+BEGIN_SRC jupyter-python :session llms :kernel llms_kernel :async yes :exports both
def m_step_student_t(x, log_gamma, mu_prev, sigma_prev, nu_prev,
                     nu_lo=3.0, nu_hi=30.0):
    # ECME M-step.
    # nu_lo=3 is a standard floor for t-mixtures: ν<3 gives infinite-variance
    # states that absorb both calm and extreme observations into one
    # component, defeating the regime-detection objective even though they
    # raise the likelihood. nu_hi=30 caps the upper end: above ~30 a t and a
    # Gaussian are visually indistinguishable, and unbounded ν drifts EM into
    # a de-facto Gaussian fit.
    gamma = np.exp(log_gamma)                            # (T,K)
    K = gamma.shape[1]

    z2 = ((x[:, None] - mu_prev[None, :]) / sigma_prev[None, :]) ** 2
    w = (nu_prev[None, :] + 1.0) / (nu_prev[None, :] + z2)   # (T,K) precision weights
    gw = gamma * w                                           # γ × w

    Nk = gamma.sum(axis=0)                                   # (K,)
    Sw = gw.sum(axis=0)                                      # (K,)

    mu = (gw * x[:, None]).sum(axis=0) / Sw
    diff_sq = (x[:, None] - mu[None, :]) ** 2
    sigma = np.sqrt((gw * diff_sq).sum(axis=0) / Nk) + 1e-6

    # ν update (Liu & Rubin ECME). Recompute w at the new (μ, σ) using
    # ν_prev, then solve  log(ν/2) - ψ(ν/2) + 1 + C = 0  for each state,
    # where  C = (1/Nk) Σ γ_t (log w_t - w_t)
    #          + ψ((ν_prev+1)/2) - log((ν_prev+1)/2).
    # The bracketed term is the E[log U] correction that comes from the
    # gamma-normal conjugate posterior; small but stabilises the fixed point.
    z2_new = ((x[:, None] - mu[None, :]) / sigma[None, :]) ** 2
    w_new = (nu_prev[None, :] + 1.0) / (nu_prev[None, :] + z2_new)
    const = ((gamma * (np.log(w_new) - w_new)).sum(axis=0) / Nk)         # (K,)
    const = const + digamma((nu_prev + 1) / 2.0) - np.log((nu_prev + 1) / 2.0)

    nu = np.empty(K)
    for i in range(K):
        c = const[i]
        def f(v, c=c):
            return np.log(v / 2.0) - digamma(v / 2.0) + 1.0 + c
        # f is monotone decreasing in ν over the search range; bracket and bisect.
        try:
            lo_val, hi_val = f(nu_lo), f(nu_hi)
            if lo_val * hi_val > 0:
                # no sign change → clamp to whichever bound the equation favours
                nu[i] = nu_lo if abs(lo_val) < abs(hi_val) else nu_hi
            else:
                nu[i] = brentq(f, nu_lo, nu_hi, xtol=1e-3)
        except Exception:
            nu[i] = nu_prev[i]
    return mu, sigma, nu
#+END_SRC

** Generic Baum-Welch driver + restart wrapper

#+BEGIN_SRC jupyter-python :session llms :kernel llms_kernel :async yes :exports both
@dataclass
class HMMFit:
    log_pi: np.ndarray
    log_A: np.ndarray
    mu: np.ndarray
    sigma: np.ndarray
    nu: np.ndarray            # all-inf for Gaussian
    log_gamma: np.ndarray
    log_likelihood: float
    n_iter: int
    converged: bool
    family: str               # "gaussian" or "student_t"


def viterbi(log_pi, log_A, log_B):
    """Most-likely state sequence (1D int array of length T)."""
    T, K = log_B.shape
    delta = np.full((T, K), -np.inf)
    psi = np.zeros((T, K), dtype=np.int64)
    delta[0] = log_pi + log_B[0]
    for t in range(1, T):
        scores = delta[t - 1][:, None] + log_A             # (K_prev, K_curr)
        psi[t] = scores.argmax(axis=0)
        delta[t] = scores.max(axis=0) + log_B[t]
    path = np.empty(T, dtype=np.int64)
    path[-1] = int(delta[-1].argmax())
    for t in range(T - 2, -1, -1):
        path[t] = psi[t + 1, path[t + 1]]
    return path


def baum_welch(x, K, family, rng, max_iter=1000, tol=1e-5, nu0=8.0, jitter=0.0):
    mu, sigma, pi, A = kmeans_init(x, K, rng, jitter=jitter)
    log_pi = np.log(pi + 1e-12)
    log_A = np.log(A + 1e-12)
    nu = np.full(K, nu0) if family == "student_t" else np.full(K, np.inf)

    prev_ll = -np.inf
    converged = False
    it = 0
    for it in range(max_iter):
        if family == "gaussian":
            log_B = gaussian_log_emissions(x, mu, sigma)
        else:
            log_B = student_t_log_emissions(x, mu, sigma, nu)
        log_gamma, log_xi, ll = forward_backward(log_pi, log_A, log_B)
        log_pi, log_A = m_step_init_trans(log_gamma, log_xi)
        if family == "gaussian":
            mu, sigma = m_step_gaussian(x, log_gamma)
        else:
            mu, sigma, nu = m_step_student_t(x, log_gamma, mu, sigma, nu)
        if abs(ll - prev_ll) < tol:
            converged = True
            it += 1
            break
        prev_ll = ll
    return HMMFit(log_pi, log_A, mu, sigma, nu, log_gamma, ll, it, converged, family)


def fit_with_restarts(x, K, family, n_restarts=5, master_seed=RNG_SEED, **kw):
    """First restart uses the clean k-means init; subsequent restarts jitter μ
    so the EM trajectory explores more than one basin on 1D data."""
    best = None
    master = np.random.default_rng(master_seed)
    for r in range(n_restarts):
        rng = np.random.default_rng(master.integers(1 << 31))
        jitter = 0.0 if r == 0 else 0.15
        try:
            fit = baum_welch(x, K, family, rng, jitter=jitter, **kw)
        except Exception as e:
            print(f"  restart {r} ({family}) failed: {e}")
            continue
        if best is None or fit.log_likelihood > best.log_likelihood:
            best = fit
    return best

REGIME_NAMES = ("bear", "ranging", "bull")

def regime_posterior(fit):
    """Return (T,3) array of posteriors in (bear, ranging, bull) order, and the
    state-index permutation used."""
    order = np.argsort(fit.mu)      # ascending μ → bear, ranging, bull
    gamma = np.exp(fit.log_gamma)
    return gamma[:, order], order

#+RESULTS:

* Pass criterion — 2022 only

Per the Phase 0 decision gate:

1. Median Student's-t bear-posterior over 2022 ≥ 0.6
2. Regime flips over 2022 ≤ 6
3. Gap vs Gaussian ≥ 0.15 OR flip-count reduction ≥ 30%

**Flip counting:** the plan reads "regime flips per calendar year". The
natural one-state-per-day labelling for an HMM is the Viterbi most-likely
path, not argmax-of-marginal-posterior — argmax of the marginal flickers
on day-to-day uncertainty, whereas Viterbi pays the transition cost. We
count flips on the Viterbi path. (Argmax counts are reported alongside
as a sanity check.)

#+BEGIN_SRC jupyter-python :session llms :kernel llms_kernel :async yes :exports both
def viterbi_states_canonical(fit, order):
    """Run Viterbi on the fitted HMM and remap to canonical bear/ranging/bull."""
    if fit.family == "gaussian":
        log_B = gaussian_log_emissions(returns, fit.mu, fit.sigma)
    else:
        log_B = student_t_log_emissions(returns, fit.mu, fit.sigma, fit.nu)
    raw = viterbi(fit.log_pi, fit.log_A, log_B)
    remap = np.empty_like(order)
    for canonical_idx, raw_idx in enumerate(order):
        remap[raw_idx] = canonical_idx
    return remap[raw]


def count_flips(state_seq):
    return int((np.diff(state_seq) != 0).sum())


g_states = viterbi_states_canonical(g_fit, g_order)
t_states = viterbi_states_canonical(t_fit, t_order)


def evaluate_2022(g_post, t_post, g_states, t_states, idx_2022):
    g22_post, t22_post = g_post[idx_2022], t_post[idx_2022]
    g22_st, t22_st = g_states[idx_2022], t_states[idx_2022]

    g_bear_med = float(np.median(g22_post[:, 0]))
    t_bear_med = float(np.median(t22_post[:, 0]))

    g_flips = count_flips(g22_st)
    t_flips = count_flips(t22_st)

    # Sanity check: argmax-of-marginal flips, the noisier metric.
    g_flips_arg = int((np.diff(g22_post.argmax(axis=1)) != 0).sum())
    t_flips_arg = int((np.diff(t22_post.argmax(axis=1)) != 0).sum())

    gap = t_bear_med - g_bear_med
    flip_reduction = (g_flips - t_flips) / g_flips if g_flips > 0 else 0.0

    crit1 = t_bear_med >= 0.6
    crit2 = t_flips <= 6
    crit3 = (gap >= 0.15) or (flip_reduction >= 0.30)

    return {
        "g_bear_median":               g_bear_med,
        "t_bear_median":               t_bear_med,
        "g_viterbi_flips_2022":        g_flips,
        "t_viterbi_flips_2022":        t_flips,
        "g_argmax_flips_2022":         g_flips_arg,
        "t_argmax_flips_2022":         t_flips_arg,
        "gap":                         gap,
        "flip_reduction_pct":          100 * flip_reduction,
        "crit1_bear_med_ge_0.6":       crit1,
        "crit2_flips_le_6":            crit2,
        "crit3_gap_or_flipreduction":  crit3,
        "verdict": "PASS" if (crit1 and crit2 and crit3) else (
            "BORDERLINE" if sum([crit1, crit2, crit3]) == 2 else "FAIL"
        ),
    }


result = evaluate_2022(g_post, t_post, g_states, t_states, idx_2022)
for k, v in result.items():
    print(f"  {k:32s} {v}")

#+RESULTS:

* Negative result — writeup

** Hypothesis under test

Phase 0 was set up to falsify a single, narrowly-scoped claim before
committing to seven days of dashboard engineering:

#+BEGIN_QUOTE
A 3-state Student's-t HMM, fit to BTC daily log-returns over the
two-year window wrapping the 2022 bear, will identify 2022 as a
sustained bear regime — substantively better than a Gaussian HMM
with the same shape, init, and restart strategy.
#+END_QUOTE

If the claim survives Phase 0 the rest of the plan rolls forward
(refactor → four-window comparison → dashboard → blog). If it fails,
the cleaner action is to stop, write up the negative finding, and
revise the strategic plan before spending the dashboard days.

** What we found

The hypothesis does not survive. Concretely, on BTCUSDT daily
2021-06-02 → 2023-06-30 (759 observations), fit under a shared
forward-backward / Baum-Welch EM with 5 jittered restarts:

| Metric (over calendar 2022)            | Gaussian | Student's-t |
|----------------------------------------|---------:|------------:|
| Median bear-state posterior            |     0.15 |        0.03 |
| Viterbi regime flips                   |       77 |           0 |
| Argmax-of-marginal flips (sanity)      |       85 |          28 |
| Fraction of 2022 in canonical "bear"   |       9% |          0% |
| Fraction of 2022 in canonical "ranging"|      25% |        100% |
| Fraction of 2022 in canonical "bull"   |      66% |          0% |

The Student's-t Viterbi path puts **100 % of the 2022 calendar year
into the "ranging" state**, and a similar 100 % into the same single
state across the bull leg (2021-06 → 2021-10) and the recovery
(2023-01 → 2023-06). It is not detecting regimes; it is fitting a
single fat-tailed component to everything.

The rule-based pass criterion in §"Pass criterion" returns
~BORDERLINE~ because two of three flags fire — but on inspection
both passing flags are artefacts:

- ~crit2 (flips ≤ 6)~ passes because Viterbi assigns one state to
  the whole year. Zero flips reflects degenerate segmentation, not
  regime stability.
- ~crit3 (gap ≥ 0.15 OR flip-reduction ≥ 30 %)~ passes only via the
  flip-reduction limb, which inherits the same degeneracy. The
  posterior-gap limb is actually negative: Student's-t bear-median
  is *lower* than Gaussian's, by 0.13.

Discounting both artefacts the substantive verdict is FAIL.

** Why it fails — root cause

BTC daily log-returns are heavy-tailed (sample excess kurtosis ≈ 3.0)
but the heavy tails are not regime-dependent in the way the plan
implicitly assumed. The fitted ν̂ comes back at ~4 for every state
under bounds [3, 30] — the EM prefers a uniformly fat-tailed solution
to a "Gaussian bull plus fat-tailed crash" decomposition.

More fundamentally, the regime structure the plan wants to detect
("2022 was a sustained bear") lives at the *cumulative-price* level,
not the *per-day-return* level. The σ of 2022 daily returns (~3.5 %)
is indistinguishable from 2021's; the drift differential is on the
order of 0.4 % / day, which is well inside one-year sampling noise.
A μ-sort labelling rule applied to a K=3 HMM fit to raw returns
therefore binds the "bear" label to the small high-σ-strongly-
negative-μ component that fires on crash *days* (LUNA, Celsius,
FTX), not on bear *months*.

In short: **raw daily returns are the wrong feature for sustained
regime detection on this asset and window.** The model is not
broken; the model is correctly reporting that 2022 daily returns
look like a fat-tailed walk near zero, the same way every other
post-2021 period does.

** Implementation notes that survive

The notebook itself is reusable independent of the negative verdict:

1. Forward-backward and Viterbi in log-space (numerically stable).
2. Shared Baum-Welch driver that swaps between Gaussian and
   Student's-t emissions, with identical init, tolerance, and
   restart strategy — the apples-to-apples comparison the plan
   asked for.
3. Liu-and-Rubin ECME ν-update with the E[log U] correction
   term and bounds [3, 30] to block both the ν→2+ degenerate
   solution and the ν→∞ Gaussian-collapse.
4. 1D-aware restart jitter (k-means alone is essentially
   deterministic in one dimension, so naive restarts clone each
   other).
5. Viterbi-based flip counting with argmax-of-marginal counts
   reported alongside as a sanity check.

These are the bits worth keeping when the work is reframed as a
methodology post.

** Recommended next direction

Not the planned dashboard. Of the response options the plan
anticipated, two are coherent given this result:

1. **Lessons-learned writeup** (~ half a day, "honest > polished").
   Frame the post around what the negative finding teaches: at
   daily frequency, on a window this narrow, neither Gaussian nor
   Student's-t with μ-sort labelling separates a sustained-bear
   regime from the modal "fat-tailed walk near zero". This is a
   real contribution — most regime-detection posts ship the
   pretty figure without checking that the "bear" label binds
   to what a human would call a bear.

2. **Reframe and retry** before writing — but with an explicit
   change of *feature*, not just *parameter*. Candidate features
   that put regime structure into per-observation space:
   rolling-window realised volatility, drawdown from rolling max,
   |returns| transformed by a sigmoid, or a two-channel observation
   (return, |return|). This deviates from the plan as written but
   is the smallest methodology change that could plausibly save
   the headline claim.

The strategic call between the two is a user decision. The plan's
own "Risks" section anticipates this exact branching at the Day 1
gate.
