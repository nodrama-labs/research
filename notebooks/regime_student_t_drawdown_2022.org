* Description

Heavy-tail HMM track using *drawdown from rolling max*::

Observation:

   d_t = log(p_t) − max_{s ≤ t} log(p_s)

Properties:
- d_t ≤ 0 by construction; equality at every running-ATH.
- Mean-reverts toward 0 in uptrends as new highs are made.
- Sits deeply negative across sustained drawdowns; the longer the
  drawdown, the more time d_t spends far from 0.
- *Causal*: the running max uses only data up to t, so this
  observation has no lookahead.
- Economically natural: drawdown *is* what defines a bear in trader
  vocabulary.

Window: 2021-06-01 → 2023-06-30. Asset: BTCUSDT daily.
Models compared: Gaussian and Student's-t HMMs at K ∈ {2, 3}.
Pass criterion: median bear-posterior over
2022 ≥ 0.6, flips ≤ 6, Student-t-vs-Gaussian gap ≥ 0.15 OR flip
reduction ≥ 30 %.

Hypothesis: bear regimes light up as "d_t sits far below 0 for many
days". K = 2 should give a clean ("near-ATH", "drawdown") split. K = 3
should add an intermediate state.

Risks:
1. *Bounded support*. d_t ≤ 0 hard. A Gaussian or Student's-t emission
   can put mass above 0 where the true density is zero — model
   mis-specification. We expect this to inflate σ of the "near-ATH"
   state to absorb mass piled up at zero. Diagnostic: if μ̂ of the
   bull state is well above 0 (i.e. the fit needs slack to cover
   below-zero observations), the misspecification is biting.
2. *Boundary at t = 0*. The first observation has d_0 = 0 by
   construction, so the initial state is forced "near-ATH"
   independent of context. Minor; the running max quickly catches up.
3. *Memory*. Rolling max is monotone non-decreasing, so consecutive
   d_t values are strongly serially correlated through the max.
   This violates the conditional independence assumption of the HMM
   emission. Should not be fatal — most financial HMMs make the same
   approximation — but worth flagging.

* HMM core — shared forward-backward + Baum-Welch

Standard log-space EM for HMMs with Gaussian or Student's-t emissions,

** Forward-backward in log-space

#+BEGIN_SRC jupyter-python :session llms :kernel llms_kernel :async yes :exports both
def forward_backward(log_pi, log_A, log_B):
    T, K = log_B.shape
    log_alpha = np.full((T, K), -np.inf)
    log_beta = np.full((T, K), -np.inf)

    log_alpha[0] = log_pi + log_B[0]
    for t in range(1, T):
        log_alpha[t] = logsumexp(log_alpha[t - 1][:, None] + log_A, axis=0) + log_B[t]

    log_beta[T - 1] = 0.0
    for t in range(T - 2, -1, -1):
        log_beta[t] = logsumexp(log_A + log_B[t + 1][None, :] + log_beta[t + 1][None, :], axis=1)

    log_likelihood = logsumexp(log_alpha[T - 1])
    log_gamma = log_alpha + log_beta - log_likelihood
    log_xi = (
        log_alpha[:-1, :, None]
        + log_A[None, :, :]
        + log_B[1:, None, :]
        + log_beta[1:, None, :]
        - log_likelihood
    )
    return log_gamma, log_xi, log_likelihood
#+END_SRC

#+RESULTS:

** Emission log-pdfs

#+BEGIN_SRC jupyter-python :session llms :kernel llms_kernel :async yes :exports both
def gaussian_log_emissions(x, mu, sigma):
    z = (x[:, None] - mu[None, :]) / sigma[None, :]
    return -0.5 * np.log(2 * np.pi) - np.log(sigma)[None, :] - 0.5 * z * z


def student_t_log_emissions(x, mu, sigma, nu):
    K = len(mu)
    out = np.empty((len(x), K))
    for i in range(K):
        out[:, i] = student_t.logpdf(x, df=nu[i], loc=mu[i], scale=sigma[i])
    return out
#+END_SRC

#+RESULTS:

** Init (shared) and transition / initial-state M-step

#+BEGIN_SRC jupyter-python :session llms :kernel llms_kernel :async yes :exports both
def kmeans_init(x, K, rng, jitter=0.0):
    km = KMeans(n_clusters=K, n_init=10, random_state=int(rng.integers(1 << 31)))
    labels = km.fit_predict(x.reshape(-1, 1))
    mu = np.array([x[labels == i].mean() for i in range(K)])
    sigma = np.array([x[labels == i].std(ddof=0) + 1e-6 for i in range(K)])
    pi = np.array([(labels == i).mean() for i in range(K)])
    if jitter > 0:
        mu = mu + rng.normal(0, jitter * x.std(), size=K)
    A = np.full((K, K), 0.05 / (K - 1)) if K > 1 else np.ones((1, 1))
    np.fill_diagonal(A, 0.95)
    return mu, sigma, pi, A


def m_step_init_trans(log_gamma, log_xi):
    log_pi = log_gamma[0] - logsumexp(log_gamma[0])
    log_A = logsumexp(log_xi, axis=0) - logsumexp(log_xi, axis=(0, 2))[:, None]
    return log_pi, log_A
#+END_SRC

#+RESULTS:

** Gaussian M-step

#+BEGIN_SRC jupyter-python :session llms :kernel llms_kernel :async yes :exports both
def m_step_gaussian(x, log_gamma):
    gamma = np.exp(log_gamma)
    Nk = gamma.sum(axis=0)
    mu = (gamma * x[:, None]).sum(axis=0) / Nk
    diff_sq = (x[:, None] - mu[None, :]) ** 2
    sigma = np.sqrt((gamma * diff_sq).sum(axis=0) / Nk) + 1e-6
    return mu, sigma
#+END_SRC

#+RESULTS:

** Student's-t M-step (ECME)

#+BEGIN_SRC jupyter-python :session llms :kernel llms_kernel :async yes :exports both
def m_step_student_t(x, log_gamma, mu_prev, sigma_prev, nu_prev,
                     nu_lo=3.0, nu_hi=30.0):
    gamma = np.exp(log_gamma)
    K = gamma.shape[1]

    z2 = ((x[:, None] - mu_prev[None, :]) / sigma_prev[None, :]) ** 2
    w = (nu_prev[None, :] + 1.0) / (nu_prev[None, :] + z2)
    gw = gamma * w

    Nk = gamma.sum(axis=0)
    Sw = gw.sum(axis=0)

    mu = (gw * x[:, None]).sum(axis=0) / Sw
    diff_sq = (x[:, None] - mu[None, :]) ** 2
    sigma = np.sqrt((gw * diff_sq).sum(axis=0) / Nk) + 1e-6

    z2_new = ((x[:, None] - mu[None, :]) / sigma[None, :]) ** 2
    w_new = (nu_prev[None, :] + 1.0) / (nu_prev[None, :] + z2_new)
    const = ((gamma * (np.log(w_new) - w_new)).sum(axis=0) / Nk)
    const = const + digamma((nu_prev + 1) / 2.0) - np.log((nu_prev + 1) / 2.0)

    nu = np.empty(K)
    for i in range(K):
        c = const[i]
        def f(v, c=c):
            return np.log(v / 2.0) - digamma(v / 2.0) + 1.0 + c
        try:
            lo_val, hi_val = f(nu_lo), f(nu_hi)
            if lo_val * hi_val > 0:
                nu[i] = nu_lo if abs(lo_val) < abs(hi_val) else nu_hi
            else:
                nu[i] = brentq(f, nu_lo, nu_hi, xtol=1e-3)
        except Exception:
            nu[i] = nu_prev[i]
    return mu, sigma, nu
#+END_SRC

#+RESULTS:

** Generic Baum-Welch driver + restart wrapper

#+BEGIN_SRC jupyter-python :session llms :kernel llms_kernel :async yes :exports both
@dataclass
class HMMFit:
    log_pi: np.ndarray
    log_A: np.ndarray
    mu: np.ndarray
    sigma: np.ndarray
    nu: np.ndarray
    log_gamma: np.ndarray
    log_likelihood: float
    n_iter: int
    converged: bool
    family: str
    K: int


def viterbi(log_pi, log_A, log_B):
    T, K = log_B.shape
    delta = np.full((T, K), -np.inf)
    psi = np.zeros((T, K), dtype=np.int64)
    delta[0] = log_pi + log_B[0]
    for t in range(1, T):
        scores = delta[t - 1][:, None] + log_A
        psi[t] = scores.argmax(axis=0)
        delta[t] = scores.max(axis=0) + log_B[t]
    path = np.empty(T, dtype=np.int64)
    path[-1] = int(delta[-1].argmax())
    for t in range(T - 2, -1, -1):
        path[t] = psi[t + 1, path[t + 1]]
    return path


def baum_welch(x, K, family, rng, max_iter=1000, tol=1e-5, nu0=8.0, jitter=0.0):
    mu, sigma, pi, A = kmeans_init(x, K, rng, jitter=jitter)
    log_pi = np.log(pi + 1e-12)
    log_A = np.log(A + 1e-12)
    nu = np.full(K, nu0) if family == "student_t" else np.full(K, np.inf)

    prev_ll = -np.inf
    converged = False
    it = 0
    for it in range(max_iter):
        if family == "gaussian":
            log_B = gaussian_log_emissions(x, mu, sigma)
        else:
            log_B = student_t_log_emissions(x, mu, sigma, nu)
        log_gamma, log_xi, ll = forward_backward(log_pi, log_A, log_B)
        log_pi, log_A = m_step_init_trans(log_gamma, log_xi)
        if family == "gaussian":
            mu, sigma = m_step_gaussian(x, log_gamma)
        else:
            mu, sigma, nu = m_step_student_t(x, log_gamma, mu, sigma, nu)
        if abs(ll - prev_ll) < tol:
            converged = True
            it += 1
            break
        prev_ll = ll
    return HMMFit(log_pi, log_A, mu, sigma, nu, log_gamma, ll, it, converged, family, K)


def fit_with_restarts(x, K, family, n_restarts=5, master_seed=RNG_SEED, **kw):
    best = None
    master = np.random.default_rng(master_seed)
    for r in range(n_restarts):
        rng = np.random.default_rng(master.integers(1 << 31))
        jitter = 0.0 if r == 0 else 0.15
        try:
            fit = baum_welch(x, K, family, rng, jitter=jitter, **kw)
        except Exception as e:
            print(f"  restart {r} ({family}, K={K}) failed: {e}")
            continue
        if best is None or fit.log_likelihood > best.log_likelihood:
            best = fit
    return best

** State → regime labelling

Drawdown reverses the μ-sort intuition versus log-return: here the
"bear" state has the most negative μ (deepest sustained drawdown) and
the "bull" state has μ closest to 0 (price at or near rolling ATH).
K = 2 → (bear, bull); K = 3 → (bear, ranging, bull) by ascending μ.

def regime_posterior(fit):
    order = np.argsort(fit.mu)
    gamma = np.exp(fit.log_gamma)
    return gamma[:, order], order

#+RESULTS:

* Pass criterion — 2022 only

Flip count from Viterbi for the primary metric, argmax-of-marginal as
a sanity check. crit3 compares Student-t to Gaussian at the same K.

#+BEGIN_SRC jupyter-python :session llms :kernel llms_kernel :async yes :exports both
def viterbi_states_canonical(fit, order):
    if fit.family == "gaussian":
        log_B = gaussian_log_emissions(obs, fit.mu, fit.sigma)
    else:
        log_B = student_t_log_emissions(obs, fit.mu, fit.sigma, fit.nu)
    raw = viterbi(fit.log_pi, fit.log_A, log_B)
    remap = np.empty_like(order)
    for canonical_idx, raw_idx in enumerate(order):
        remap[raw_idx] = canonical_idx
    return remap[raw]


def count_flips(state_seq):
    return int((np.diff(state_seq) != 0).sum())


viterbi_states = {key: viterbi_states_canonical(fit, orders[key])
                  for key, fit in fits.items()}


def evaluate_2022(K):
    g_post = posteriors[("gaussian", K)][idx_2022]
    t_post = posteriors[("student_t", K)][idx_2022]
    g_st = viterbi_states[("gaussian", K)][idx_2022]
    t_st = viterbi_states[("student_t", K)][idx_2022]

    g_bear_med = float(np.median(g_post[:, 0]))
    t_bear_med = float(np.median(t_post[:, 0]))

    g_flips = count_flips(g_st)
    t_flips = count_flips(t_st)
    g_flips_arg = int((np.diff(g_post.argmax(axis=1)) != 0).sum())
    t_flips_arg = int((np.diff(t_post.argmax(axis=1)) != 0).sum())

    gap = t_bear_med - g_bear_med
    flip_reduction = (g_flips - t_flips) / g_flips if g_flips > 0 else 0.0

    crit1_g = g_bear_med >= 0.6
    crit1_t = t_bear_med >= 0.6
    crit2_g = g_flips <= 6
    crit2_t = t_flips <= 6
    crit3 = (gap >= 0.15) or (flip_reduction >= 0.30)

    def verdict(crit1, crit2, crit3_or_solo):
        score = sum([crit1, crit2, crit3_or_solo])
        return "PASS" if score == 3 else ("BORDERLINE" if score == 2 else "FAIL")

    return {
        "K":                           K,
        "g_bear_median":               g_bear_med,
        "t_bear_median":               t_bear_med,
        "g_viterbi_flips_2022":        g_flips,
        "t_viterbi_flips_2022":        t_flips,
        "g_argmax_flips_2022":         g_flips_arg,
        "t_argmax_flips_2022":         t_flips_arg,
        "gap":                         gap,
        "flip_reduction_pct":          100 * flip_reduction,
        "crit1_g_bear_med_ge_0.6":     crit1_g,
        "crit1_t_bear_med_ge_0.6":     crit1_t,
        "crit2_g_flips_le_6":          crit2_g,
        "crit2_t_flips_le_6":          crit2_t,
        "crit3_gap_or_flipreduction":  crit3,
        "verdict_student_t_vs_g":      verdict(crit1_t, crit2_t, crit3),
        "gaussian_alone_passes_1_2":   crit1_g and crit2_g,
    }


for K in (2, 3):
    print(f"\n=== K = {K} ===")
    result = evaluate_2022(K)
    for k, v in result.items():
        print(f"  {k:34s} {v}")

* Positive result — writeup

** Hypothesis under test

Whether *drawdown from rolling max* — a strictly causal observation
that maps directly to the trader definition of a bear — produces a
clean identification of 2022 as a sustained bear regime on the
2021-06 → 2023-06 BTCUSDT window, with standard Gaussian / Student's-t
HMM emissions at K ∈ {2, 3}.

** What we found

The hypothesis survives, with a particularly clean K = 3 decomposition.

| Metric (over calendar 2022)             | Gaussian K=2 | Student-t K=2 | Gaussian K=3 | Student-t K=3 |
|-----------------------------------------|-------------:|--------------:|-------------:|--------------:|
| Median bear-state posterior             |         1.00 |          1.00 |         1.00 |          1.00 |
| Viterbi flips in 2022                   |            1 |             1 |            1 |             1 |
| Argmax-of-marginal flips (sanity)       |            1 |             1 |            1 |             1 |
| Fraction of 2022 in "bear"              |          65% |           65% |          55% |           55% |
| Fraction of 2022 in "ranging" (K=3)     |              |               |          45% |           45% |
| Fraction of 2022 in "bull"              |          35% |           35% |           0% |            0% |

The K = 3 Gaussian fit yields three states with clean monotone μ and
small σ:

| Regime  | μ̂       | σ̂      | Interpretation                                  |
|---------|--------:|-------:|-------------------------------------------------|
| bear    |  −1.21  |  0.12  | ~70 % off running ATH, tight cluster            |
| ranging |  −0.69  |  0.20  | ~50 % off ATH, wider — local rallies / declines |
| bull    |  −0.13  |  0.11  | ~12 % off ATH, tight — near running highs       |

Per-window composition (K = 3 Gaussian), all four windows:

| Window                          | bear | ranging | bull |
|---------------------------------|-----:|--------:|-----:|
| 2021-06 → 2021-10 (bull leg)    |   0% |      0% | 100% |
| 2021-11 → 2022-12 (full bear)   |  47% |     39% |  13% |
| 2022 (calendar year)            |  55% |     45% |   0% |
| 2023-01 → 2023-06 (recovery)    |  41% |     59% |   0% |

Every window's composition matches the human label by eye:
- The bull leg is 100 % bull.
- 2022 is 100 % (bear + ranging) — no bull contamination.
- The 2023 recovery is dominantly ranging (price still well below
  ATH but rising off the cycle low), with the lingering bear-state
  presence reflecting Jan-Feb 2023 still being near the bottom.

K = 2 also passes the pass criterion but absorbs all of 2023 into
"bear" (since d_t is still deeply negative throughout) and gives a
binary near-ATH-vs-deep-drawdown split that loses the recovery
nuance. K = 3 is the right K for this feature.

Rule-based ~BORDERLINE~ on crit3 because Student-t and Gaussian fits
are essentially identical (ν̂ saturates at the upper bound 30 in every
state), but the substantive verdict is PASS on the regime-identification
question.

** Why it works — root cause

Drawdown from rolling max is a strong cumulative-level feature for
regime detection because:

1. *Causal by construction.* The running max uses only data up to t,
   so there's no hindsight in the observation — no rolling-window or
   centred-z-score caveat to defend.
2. *Bull state means what it says.* "Bull" = "near running ATH". The
   2023-H1 recovery (price rising but still well below the cycle ATH)
   correctly classifies as ranging — not bear, not bull. A
   level-anchored feature like full-window z-scored log-price gets
   this wrong: "bull" there means "above the window mean", which
   penalises early recoveries.
3. *Cleaner state separation.* In K = 3, the (μ, σ) for the three
   states form a near-evenly-spaced grid on the d_t axis (−1.21,
   −0.69, −0.13) with comparable σ across states. EM doesn't have
   to fight a single dominant distribution for variance share.

The saturation of ν at 30 confirms that heavy-tail emissions are
unnecessary at this feature scale. Drawdown is bounded above (d_t ≤ 0)
and varies on the order of σ = 0.1–0.2 log-units; no observations are
far enough from any state's μ to warrant Student-t tails.

** Caveats

1. *Gaussian violates the d_t ≤ 0 hard bound.* For the "bull" state
   (μ = −0.13, σ = 0.11), the Gaussian assigns ~12 % probability mass
   above 0, where the true density is exactly zero. The fit still
   works because the bound is monotone (no observations actually fall
   above 0, so the bound never bites in the likelihood). For a
   different window where d_t spends more time near 0 (a fresh bull
   making frequent new ATHs), this misspecification would inflate the
   σ of the bull state. A principled alternative is a truncated
   Gaussian or a beta on −d_t — neither is necessary here.
2. *t = 0 boundary.* d_0 = 0 by construction, so the very first
   observation looks "bull" regardless of context. Immaterial: by
   day ~30 the running max has caught the cycle peak and the
   classification is data-driven.
3. *Strong serial correlation through the max.* The HMM emission
   conditional-independence assumption is violated. This is the
   standard approximation in regime-detection on cumulative
   features and is not fatal here, but it means the LL values
   should be read as relative-fit-quality not as a calibrated
   probability.
4. *Window-specific interpretation.* "Ranging" here is a specific
   thing: somewhere between 30–60 % off the running ATH and either
   chopping sideways or rallying. In a window dominated by a long
   sideways consolidation near ATH, "ranging" would absorb
   different days. The decomposition is conditional on the
   2021-06 → 2023-06 dynamics.

** Implementation notes

The whole observation transform is a single line:
~df["drawdown"] = df["log_price"] − df["log_price"].cummax()~.
Conservation laws hold throughout: LL monotone non-decreasing under
EM, ν stays within [3, 30], Gaussian converges within ~10 iterations,
Student-t within ~30.

** Recommended next direction

Standalone result: "drawdown from rolling max is the right observation
for HMM-based regime detection in BTC, and a K = 3 Gaussian fit is
sufficient". Three coherent rolls-forward, ranked by code reuse vs
expected information gain:

1. *Longer window, K = 4.* Koki, Leonardos & Piliouras (2020) find that
   BTC daily-return regime structure resolves to 4 states (bear /
   bull-high-vol / bull-low-vol / calm-auxiliary), with K = 3 being
   structurally one short. With drawdown as the observation, K = 4
   might separate the bear into "approaching ATH from below" vs "deep
   capitulation" sub-states, or split the ranging state. Cheap
   (one-line K change); informative as a sanity check that the K = 3
   ceiling isn't an artefact of the short window. Recommended next.

2. *Bivariate (drawdown, log-return).* Adds the per-day signal back
   in as a second channel. The latent state would now bind to both
   "where am I in the cycle" (drawdown) and "what's happening today"
   (return), which is the natural extension toward the multivariate
   observation vector (return, realised volatility, volume) used by
   Malekinezhad & Rafati (2026) on 4-hour BTC. Needs a 2-D Gaussian
   M-step and a covariance parameterisation choice (diagonal vs
   full). ~half day of code.

3. *NH-HMM with VIX in transitions.* Koki et al. (2020) report VIX as
   the universal transition predictor across BTC/ETH/XRP (posterior
   inclusion probability 1.00), and Pakštaitė et al. (2025) confirm
   the value of time-varying transitions in a Bayesian MCMC NH-HMM
   on BTC daily log-price. Lower marginal value now that the
   homogeneous fit is this clean, but the literature-aligned next
   step beyond options 1–2.

My recommendation: option 1. It's the smallest possible change from
this passing recipe and directly tests the Koki et al. (2020)
structural claim on our own data and our own observation.
