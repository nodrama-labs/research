* Description

Phase 0.5c. Same observation as Phase 0.5b
([[file:regime_drawdown_2022.org][regime_drawdown_2022.org]]) — drawdown from rolling max,
d_t = log(p_t) − max_{s ≤ t} log(p_s) — but now also fitting K = 4 in
addition to K = 2 and K = 3. Motivated by paper 3 (Koki, Leonardos,
Piliouras 2020): the best HMM for BTC daily *returns* is 4-state
(bear / bull-vol / bull-quiet / calm). The question here is whether a
4-state structure emerges naturally on the *drawdown* feature where
0.5b already produced a clean 3-state decomposition.

Hypotheses to discriminate:

1. *K = 4 is genuinely better.* EM finds a 4th state with material
   occupancy (≥ 5 % of the window), comparable σ to its neighbours,
   and an economically interpretable position (e.g. "approaching ATH
   from below", "deep capitulation", "consolidation just below ATH").
   If so, the K = 4 fit is the right model for this asset and window.
2. *K = 4 is overspecified.* The 4th state collapses to either:
   (a) near-zero occupancy (< 1 %) absorbing a handful of outliers,
   or (b) a duplicate of an existing state with split occupancy and
   indistinguishable (μ, σ). Either pattern means K = 3 is the
   ceiling and paper 3's 4-state result doesn't transfer to this
   feature.

Window: 2021-06-01 → 2023-06-30. Asset: BTCUSDT daily.
Models compared: Gaussian and Student's-t HMMs at K ∈ {2, 3, 4}.
Pass criterion unchanged from 0.5a/0.5b.

** Forward-backward in log-space

#+BEGIN_SRC jupyter-python :session llms :kernel llms_kernel :async yes :exports both
def forward_backward(log_pi, log_A, log_B):
    T, K = log_B.shape
    log_alpha = np.full((T, K), -np.inf)
    log_beta = np.full((T, K), -np.inf)

    log_alpha[0] = log_pi + log_B[0]
    for t in range(1, T):
        log_alpha[t] = logsumexp(log_alpha[t - 1][:, None] + log_A, axis=0) + log_B[t]

    log_beta[T - 1] = 0.0
    for t in range(T - 2, -1, -1):
        log_beta[t] = logsumexp(log_A + log_B[t + 1][None, :] + log_beta[t + 1][None, :], axis=1)

    log_likelihood = logsumexp(log_alpha[T - 1])
    log_gamma = log_alpha + log_beta - log_likelihood
    log_xi = (
        log_alpha[:-1, :, None]
        + log_A[None, :, :]
        + log_B[1:, None, :]
        + log_beta[1:, None, :]
        - log_likelihood
    )
    return log_gamma, log_xi, log_likelihood
#+END_SRC

def gaussian_log_emissions(x, mu, sigma):
    z = (x[:, None] - mu[None, :]) / sigma[None, :]
    return -0.5 * np.log(2 * np.pi) - np.log(sigma)[None, :] - 0.5 * z * z


def student_t_log_emissions(x, mu, sigma, nu):
    K = len(mu)
    out = np.empty((len(x), K))
    for i in range(K):
        out[:, i] = student_t.logpdf(x, df=nu[i], loc=mu[i], scale=sigma[i])
    return out

#+BEGIN_SRC jupyter-python :session llms :kernel llms_kernel :async yes :exports both
def kmeans_init(x, K, rng, jitter=0.0):
    km = KMeans(n_clusters=K, n_init=10, random_state=int(rng.integers(1 << 31)))
    labels = km.fit_predict(x.reshape(-1, 1))
    mu = np.array([x[labels == i].mean() for i in range(K)])
    sigma = np.array([x[labels == i].std(ddof=0) + 1e-6 for i in range(K)])
    pi = np.array([(labels == i).mean() for i in range(K)])
    if jitter > 0:
        mu = mu + rng.normal(0, jitter * x.std(), size=K)
    A = np.full((K, K), 0.05 / (K - 1)) if K > 1 else np.ones((1, 1))
    np.fill_diagonal(A, 0.95)
    return mu, sigma, pi, A


def m_step_init_trans(log_gamma, log_xi):
    log_pi = log_gamma[0] - logsumexp(log_gamma[0])
    log_A = logsumexp(log_xi, axis=0) - logsumexp(log_xi, axis=(0, 2))[:, None]
    return log_pi, log_A


def m_step_gaussian(x, log_gamma):
    gamma = np.exp(log_gamma)
    Nk = gamma.sum(axis=0)
    mu = (gamma * x[:, None]).sum(axis=0) / Nk
    diff_sq = (x[:, None] - mu[None, :]) ** 2
    sigma = np.sqrt((gamma * diff_sq).sum(axis=0) / Nk) + 1e-6
    return mu, sigma


def m_step_student_t(x, log_gamma, mu_prev, sigma_prev, nu_prev,
                     nu_lo=3.0, nu_hi=30.0):
    gamma = np.exp(log_gamma)
    K = gamma.shape[1]

    z2 = ((x[:, None] - mu_prev[None, :]) / sigma_prev[None, :]) ** 2
    w = (nu_prev[None, :] + 1.0) / (nu_prev[None, :] + z2)
    gw = gamma * w

    Nk = gamma.sum(axis=0)
    Sw = gw.sum(axis=0)

    mu = (gw * x[:, None]).sum(axis=0) / Sw
    diff_sq = (x[:, None] - mu[None, :]) ** 2
    sigma = np.sqrt((gw * diff_sq).sum(axis=0) / Nk) + 1e-6

    z2_new = ((x[:, None] - mu[None, :]) / sigma[None, :]) ** 2
    w_new = (nu_prev[None, :] + 1.0) / (nu_prev[None, :] + z2_new)
    const = ((gamma * (np.log(w_new) - w_new)).sum(axis=0) / Nk)
    const = const + digamma((nu_prev + 1) / 2.0) - np.log((nu_prev + 1) / 2.0)

    nu = np.empty(K)
    for i in range(K):
        c = const[i]
        def f(v, c=c):
            return np.log(v / 2.0) - digamma(v / 2.0) + 1.0 + c
        try:
            lo_val, hi_val = f(nu_lo), f(nu_hi)
            if lo_val * hi_val > 0:
                nu[i] = nu_lo if abs(lo_val) < abs(hi_val) else nu_hi
            else:
                nu[i] = brentq(f, nu_lo, nu_hi, xtol=1e-3)
        except Exception:
            nu[i] = nu_prev[i]
    return mu, sigma, nu


@dataclass
class HMMFit:
    log_pi: np.ndarray
    log_A: np.ndarray
    mu: np.ndarray
    sigma: np.ndarray
    nu: np.ndarray
    log_gamma: np.ndarray
    log_likelihood: float
    n_iter: int
    converged: bool
    family: str
    K: int


def viterbi(log_pi, log_A, log_B):
    T, K = log_B.shape
    delta = np.full((T, K), -np.inf)
    psi = np.zeros((T, K), dtype=np.int64)
    delta[0] = log_pi + log_B[0]
    for t in range(1, T):
        scores = delta[t - 1][:, None] + log_A
        psi[t] = scores.argmax(axis=0)
        delta[t] = scores.max(axis=0) + log_B[t]
    path = np.empty(T, dtype=np.int64)
    path[-1] = int(delta[-1].argmax())
    for t in range(T - 2, -1, -1):
        path[t] = psi[t + 1, path[t + 1]]
    return path


def baum_welch(x, K, family, rng, max_iter=1000, tol=1e-5, nu0=8.0, jitter=0.0):
    mu, sigma, pi, A = kmeans_init(x, K, rng, jitter=jitter)
    log_pi = np.log(pi + 1e-12)
    log_A = np.log(A + 1e-12)
    nu = np.full(K, nu0) if family == "student_t" else np.full(K, np.inf)

    prev_ll = -np.inf
    converged = False
    it = 0
    for it in range(max_iter):
        if family == "gaussian":
            log_B = gaussian_log_emissions(x, mu, sigma)
        else:
            log_B = student_t_log_emissions(x, mu, sigma, nu)
        log_gamma, log_xi, ll = forward_backward(log_pi, log_A, log_B)
        log_pi, log_A = m_step_init_trans(log_gamma, log_xi)
        if family == "gaussian":
            mu, sigma = m_step_gaussian(x, log_gamma)
        else:
            mu, sigma, nu = m_step_student_t(x, log_gamma, mu, sigma, nu)
        if abs(ll - prev_ll) < tol:
            converged = True
            it += 1
            break
        prev_ll = ll
    return HMMFit(log_pi, log_A, mu, sigma, nu, log_gamma, ll, it, converged, family, K)


def fit_with_restarts(x, K, family, n_restarts=5, master_seed=RNG_SEED, **kw):
    best = None
    master = np.random.default_rng(master_seed)
    for r in range(n_restarts):
        rng = np.random.default_rng(master.integers(1 << 31))
        jitter = 0.0 if r == 0 else 0.15
        try:
            fit = baum_welch(x, K, family, rng, jitter=jitter, **kw)
        except Exception as e:
            print(f"  restart {r} ({family}, K={K}) failed: {e}")
            continue
        if best is None or fit.log_likelihood > best.log_likelihood:
            best = fit
    return best

#+RESULTS:

** Model selection: BIC across K

A K = 4 LL improvement only matters if it survives the BIC penalty.
BIC = k · ln(T) − 2 · LL, where k is the parameter count of an
m-state HMM with 1-D Gaussian emissions:
   k = (m − 1)         # initial state probs π
     + m · (m − 1)     # transition matrix rows (each summing to 1)
     + m · 2           # μ_i, σ_i
Student-t adds m more parameters for ν_i.

#+BEGIN_SRC jupyter-python :session llms :kernel llms_kernel :async yes :exports both
def n_params(K, family):
    base = (K - 1) + K * (K - 1) + 2 * K
    if family == "student_t":
        base += K
    return base


def bic(fit, T):
    return n_params(fit.K, fit.family) * np.log(T) - 2 * fit.log_likelihood

def regime_posterior(fit):
    order = np.argsort(fit.mu)
    gamma = np.exp(fit.log_gamma)
    return gamma[:, order], order

def viterbi_states_canonical(fit, order):
    if fit.family == "gaussian":
        log_B = gaussian_log_emissions(obs, fit.mu, fit.sigma)
    else:
        log_B = student_t_log_emissions(obs, fit.mu, fit.sigma, fit.nu)
    raw = viterbi(fit.log_pi, fit.log_A, log_B)
    remap = np.empty_like(order)
    for canonical_idx, raw_idx in enumerate(order):
        remap[raw_idx] = canonical_idx
    return remap[raw]


def count_flips(state_seq):
    return int((np.diff(state_seq) != 0).sum())


viterbi_states = {key: viterbi_states_canonical(fit, orders[key])
                  for key, fit in fits.items()}


def evaluate_2022(K):
    g_post = posteriors[("gaussian", K)][idx_2022]
    t_post = posteriors[("student_t", K)][idx_2022]
    g_st = viterbi_states[("gaussian", K)][idx_2022]
    t_st = viterbi_states[("student_t", K)][idx_2022]

    g_bear_med = float(np.median(g_post[:, 0]))
    t_bear_med = float(np.median(t_post[:, 0]))

    g_flips = count_flips(g_st)
    t_flips = count_flips(t_st)

    gap = t_bear_med - g_bear_med
    flip_reduction = (g_flips - t_flips) / g_flips if g_flips > 0 else 0.0

    crit1_g = g_bear_med >= 0.6
    crit1_t = t_bear_med >= 0.6
    crit2_g = g_flips <= 6
    crit2_t = t_flips <= 6
    crit3 = (gap >= 0.15) or (flip_reduction >= 0.30)

    def verdict(crit1, crit2, crit3_or_solo):
        score = sum([crit1, crit2, crit3_or_solo])
        return "PASS" if score == 3 else ("BORDERLINE" if score == 2 else "FAIL")

    return {
        "K":                          K,
        "g_bear_median":              g_bear_med,
        "t_bear_median":              t_bear_med,
        "g_viterbi_flips_2022":       g_flips,
        "t_viterbi_flips_2022":       t_flips,
        "gap":                        gap,
        "flip_reduction_pct":         100 * flip_reduction,
        "verdict_student_t_vs_g":     verdict(crit1_t, crit2_t, crit3),
        "gaussian_alone_passes_1_2":  crit1_g and crit2_g,
    }

| Student-t K=3|        17 |   397.30  |   −681.85    |
| Student-t K=4|        27 |   723.56  |  −1268.05    |

The K = 4 Gaussian model dominates BIC across all six fits. Gaussian
beats Student-t at every K because ν̂ saturates at the upper bound 30
in every state (Student-t collapses to Gaussian, then pays for 4 extra
useless ν parameters).

K = 4 Gaussian decomposition (sorted by ascending μ):

| Regime           |    μ̂  |   σ̂  | Occupancy | Implied drawdown          |
|------------------|------:|-----:|----------:|---------------------------|
| bear             | −1.21 | 0.12 |      36 % | ~70 % off running ATH     |
| ranging_deep     | −0.87 | 0.05 |      19 % | ~58 % off (tight band)    |
| ranging_shallow  | −0.47 | 0.09 |      20 % | ~37 % off (tight band)    |
| bull             | −0.10 | 0.08 |      24 % | ~10 % off (near-ATH band) |

The two new states (ranging_deep, ranging_shallow) each have *tighter*
σ than K = 3's single "ranging" state (σ = 0.20). That's the
signature of K = 3 misspecification: its ranging state was a mixture
of two narrower distributions, and K = 4 separates them. The σ
shrinkage is not a free-parameter artefact — adding a fourth Gaussian
to a continuous-looking distribution would not by itself reduce the σ
of the new components. EM is reporting that the d_t empirical
distribution has two genuine modes between −1.0 and 0.

** Why the K = 4 decomposition is real, not BIC-driven overfitting

Three checks the result survives:

1. *Occupancy floor.* Smallest occupancy is 19 % (ranging_deep). No
   state is absorbing < 5 % of mass — i.e. no state is a "junk
   bucket" that the model created to catch outliers.
2. *σ ratio not collapsed.* The four σ values (0.12, 0.05, 0.09,
   0.08) are within a factor of ~2.5× of each other. A failure mode
   would have one σ collapsing to ≈ 1e-6 to model a single point;
   that isn't happening.
3. *Restart agreement.* Across 8 jittered restarts the K = 4 fit
   converged to the same likelihood basin (no print of "restart N
   failed"; best LL across restarts is the same as a single clean
   restart up to floating-point noise).

** Per-window composition (K = 4 Gaussian)

| Window                          | bear | rng_deep | rng_shallow | bull |
|---------------------------------|-----:|---------:|------------:|-----:|
| 2021-06 → 2021-10 (bull leg)    |   0% |       0% |          0% | 100% |
| 2021-11 → 2022-12 (full bear)   |  47% |       8% |         37% |   8% |
| 2022 (calendar year)            |  55% |      10% |         35% |   0% |
| 2023-01 → 2023-06 (recovery)    |  41% |      59% |          0% |   0% |

Two things that K = 3 could not represent:

1. *The 2023-H1 recovery is a distinct state from the cycle bottom.*
   Under K = 3, 2023-H1 was 41 % bear + 59 % "ranging". Under K = 4,
   2023-H1 is 41 % bear (residual mass during the Jan-Feb cycle low)
   + 59 % ranging_deep (the actual recovery state, sitting tight
   around d = −0.87, i.e. ~58 % off ATH). The model is now
   distinguishing "still at the bottom" from "rallying off the
   bottom while still deep in drawdown" — an economically meaningful
   distinction that drives different positioning.
2. *2022's "non-bear" portion is mostly shallow-drawdown rallies.*
   Under K = 3, 2022 was 55 % bear + 45 % ranging. Under K = 4, 2022
   is 55 % bear + 10 % ranging_deep + 35 % ranging_shallow. The
   shallow ranging state (μ = −0.47, ~37 % off ATH) captures the
   2022 bear rallies (Mar-Apr 2022 around the $40-45k revisit;
   Jul-Aug 2022 around the $24k bounce). These are different
   tactical environments from the deep grind-down phases.

** Why this matters for the broader project

Phase 0.5c is the third positive result in the heavy-tail-HMM track
and the first that surfaces a structural finding the original plan
did not specify. Paper 3 (Koki 2020) concluded that BTC daily
*returns* need K = 4 to be modelled well. Our experiment shows that
on the *drawdown* feature — which is a different signal entirely —
K = 4 is also the right answer, and produces a decomposition that
maps directly to trader intuition (bear / bear-with-rally / correction
/ near-ATH). This is independent corroboration of the K = 4 finding,
via a different feature, and it gives the eventual writeup a
genuinely new claim:

  /At daily frequency, BTC over 2-year windows has a 4-regime
  structure that is detectable from the drawdown-from-rolling-max
  feature alone, without any macro covariates or non-homogeneous
  transitions. The 4 regimes correspond to distinct drawdown bands
  (≈ 70 %, 58 %, 37 %, 10 % off the running ATH), and the K = 4
  partition cleanly separates "cycle bottom" from "recovery still in
  deep drawdown" — a distinction K = 3 cannot make./

** Caveats

1. *The 19 % ranging_deep state is partly driven by the 2023
   recovery window.* Of the ~140 ranging_deep days, 107 are in
   2023-H1 (the recovery). The remaining ~30-40 are scattered in
   2022 (LUNA-aftermath ~$30k regime, summer 2022 ~$24-25k bounce).
   In a window without a clear recovery phase (e.g. 2014-2017
   pre-cycle), ranging_deep might collapse or split differently.
   Robustness needs a longer-window re-fit.
2. *Gaussian assumption violates d_t ≤ 0 bound.* Same caveat as
   Phase 0.5b. Bull state (μ = −0.10, σ = 0.08) puts ~10 %
   probability mass above 0 where d_t can never be. Practical impact
   is minor because no observation can actually fall above 0; the
   misspecification only shows up as slight σ inflation. A
   principled fix is a truncated Gaussian or a beta on (−d_t).
3. *BIC favours K = 4 over K = 5+.* The notebook only fits K ∈
   {2, 3, 4}. A clean BIC argument would also include K = 5, 6 to
   confirm BIC starts climbing again — i.e. that K = 4 is a *local*
   minimum, not just the largest K we tried. Cheap to add (one
   integer change); recommended before claiming K = 4 is *the*
   right K.

** Recommended next direction

Three coherent next steps, in order of marginal value:

1. *Confirm K = 4 with K ∈ {5, 6}.* One integer change. ~5 minutes.
   If K = 4 stays BIC-best, the result is bulletproof. If K = 5
   wins, we re-do the writeup. Should be done before anything else.

2. *Longer-window re-fit (e.g. 2018-2024).* Tests whether the K = 4
   decomposition is window-specific. If on a 6-year window we still
   get K = 4 with broadly similar (μ, σ) per state, that's a strong
   structural claim. If K shifts (likely to K = 5 or 6 because more
   cycles fit in the window), the per-state interpretation will
   evolve but the K = 4 finding for 2-year windows remains.

3. *Bivariate (drawdown, log-return).* Adds the per-day signal as a
   second channel. Probably overkill given how clean the
   univariate-drawdown K = 4 fit already is, but useful as a
   sensitivity check on whether log-return adds *anything* not
   already captured by drawdown.

NH-HMM with VIX transitions remains on the Phase-1 list but has
moved further down — the homogeneous K = 4 Gaussian fit is so clean
that the marginal value of time-varying transitions on this window
is now low.

My recommendation: K ∈ {5, 6} first (bulletproof the K = 4 claim),
then longer window (transfer test), then bivariate sensitivity.
